# Hooking Point — 기획 문서 (PRD)

## 서비스 개요
Hooking Point는 영화·드라마 편집자의 포트폴리오와, 자체 기획 중인 오리지널 IP(지식재산) 프로젝트를 함께 소개하는 웹사이트다. 단순 이력 나열형 포트폴리오가 아니라, "편집실"과 "IP 기획사"라는 두 축을 하나의 브랜드 아래 보여주는 것이 목적이며, 관리자가 개발자 개입 없이 스스로 콘텐츠를 갱신할 수 있는 구조로 설계했다.

## 해결하려는 문제
- 편집자의 실제 작품 이력이 SNS·이력서·구두 설명 등 여러 곳에 흩어져 있어, 신뢰할 수 있는 단일 창구가 없었다.
- 기획 중인 IP는 로그라인 정도만 외부에 공개하고 상세 시놉시스는 정식 미팅 전까지 비공개로 유지해야 하는 B2B 특성이 있어, 접근 제어가 필요했다.
- 정적 사이트 특성상 콘텐츠를 추가·수정할 때마다 개발자의 코드 수정이 필요했던 기존 방식은 지속 가능하지 않았다 — 관리자가 직접 갱신할 수 있어야 했다.

## 타겟 사용자
1. **방송사·제작사 관계자** — 편집자 섭외 전에 실제 작업물과 편집 접근 방식을 확인하려는 사람
2. **IP 사업 파트너·투자자** — 로그라인만 보고 관심이 생기면 로그인 후 상세 시놉시스를 요청하는 사람
3. **사이트 관리자(운영자 본인)** — 개발자 도움 없이 작품·IP·회원을 직접 관리하는 사람

## 핵심 기능 (요구사항)
- **작품 목록/상세 페이지**: 장르(영화/드라마)별 필터, 감독·제작사·출연진·시청 링크, 영화제 초청·수상 이력
- **IP 개발 페이지**: 로그라인은 누구나 공개 열람, 상세 시놉시스는 로그인 사용자에게만 노출(B2B 게이팅)
- **인증(Supabase Auth)**: 회원가입, 로그인, 아이디/비밀번호 찾기(OTP)
- **관리자 전용 대시보드**: 작품·IP 프로젝트 CRUD, 회원 권한(관리자 지정/해제) 및 계정 관리 — 공개 로그인 화면과 완전히 분리된 전용 진입 화면 사용
- **자동 배포 파이프라인**: GitHub에 push하면 빌드 후 Cloudflare Pages에 자동 배포

## 데이터 모델 (Supabase / PostgreSQL)
| 테이블 | 역할 | 접근 제어 |
| --- | --- | --- |
| `works` | 작품 정보(장르, 감독, 제작사, 시놉시스, 영화제 이력, 정렬 순서 등) | 공개 읽기 |
| `ip_projects` | IP 기획 카드(제목, 장르, 단계, 로그라인) | 공개 읽기 |
| `ip_project_synopses` | IP 상세 시놉시스 | 로그인 사용자만 읽기, 관리자만 쓰기 |
| `profiles` | 회원 정보 및 역할(admin/user) | 본인·관리자만 쓰기(RLS) |

## 기술 스택
Astro(정적 사이트 생성) + Supabase(Auth·Postgres, 클라이언트 사이드 호출) + Cloudflare Pages(배포) + GitHub Actions(CI/CD) — 별도 서버 비용 없이 동적 관리 기능을 구현했다.
